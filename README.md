# incident-graph

> a GraphRAG based incident root cause analyzer.

This is a Retrieval-Augmented Generation (RAG) project using LangChain, FAISS, and Google's Gemini models. The system is designed to read text files, store their embeddings locally in a vector database, and provide a chatbot interface that can answer questions based on the ingested documents.

The project is structured with a clean, flat architecture containing a shared utility script, an offline ingestion script, and an interactive online chatbot script.

## Project Structure

*   **`generate_embeddings.py`**: The offline ingestion utility. It parses all `.txt` files inside the `./inputs` directory, chunks them, generates embeddings using Gemini, and saves the vector database to the `./faiss_index` directory.
*   **`chat_bot.py`**: The interactive chat application. It detects if a `faiss_index` database exists. If it does, the bot answers your questions strictly using the document context (RAG mode) and refuses to hallucinate if the answer is missing. If the database does not exist, it acts as a standard general-knowledge AI assistant.
*   **`utils.py`**: A shared utility file containing the `CustomGeminiEmbeddings` class, which handles chunk-by-chunk embeddings to gracefully bypass known batching limitations in the Gemini SDK.
*   **`inputs/`**: The folder where you place all your source `.txt` files (e.g., incident reports, documentation, logs).
*   **`faiss_index/`**: The local storage directory generated by the embedding script (ignored by git).

## Setup Instructions

1.  **Environment Variables**:
    Ensure you have your Google API key configured in your environment so the Gemini models can authenticate.
    ```bash
    export GOOGLE_API_KEY="your-api-key-here"
    # Alternatively, you can use GEMINI_API_KEY
    ```

2.  **Package Manager**:
    This project uses [uv](https://github.com/astral-sh/uv) for fast package management. All dependencies have been initialized inside `pyproject.toml` and `uv.lock`.

## Usage

### 1. Ingest Documents (Offline Database Generation)
Whenever you add, modify, or remove text files in the `./inputs` folder, you should rebuild your local FAISS vector database.

Run the following command:
```bash
uv run python generate_embeddings.py
```
This script will:
- Clean up any old index.
- Read all `.txt` documents in `./inputs`.
- Split them into overlapping 500-character chunks.
- Generate high-quality embeddings using `gemini-embedding-2`.
- Save the resulting FAISS index locally.

### 2. Run the Chatbot
To start asking questions about your documents, launch the interactive bot:

```bash
uv run python chat_bot.py
```
Once started, you can type your queries.
- **If `faiss_index` exists**: The bot operates in RAG mode. It will search the database and only provide answers found in your documents.
- **If `faiss_index` is missing**: The bot operates in Standard mode, acting as a general-knowledge AI without your local context.
- **Exit**: Type `exit` or `quit` to close the application.
